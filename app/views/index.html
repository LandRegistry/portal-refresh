View My Applications v1.0

This version incorporates all of the discussed changes to the design of VMA that includes headers and footers, a left hand menu, the search box, filtering, and an expanded entry etc.

  • Added filters (checkboxes)
  • Aligned search bar to the right side of the page.
  • Added 'Estimated completion' dates.
  • Added feature to sort order 'Recieved by HMLR' and 'Estimated completion'.

The left hand menu will have content that concurs with the other areas of Portal and will look similar to the other menus on other screens i.e Homepage left hand menu.

  • The font colour has been changed to green (#516715).
  • The font colour to identify the current page is black (#000000).
  • The hover background remains as green (#D4E039).

The header of the VMA will incorporate:

  • The header will contain the links “Logout” followed by “Customer Support”.
  • It was not agreed whether there would be a pop out window icon next to customer support. This will need to be checked with the agreed header / footer design.
  • The Land Registry Title and the Land Registry Logo, that if clicked, will take the user back to the Home Page.

The footer of the VMA will incorporate:

  • The four links as signed off around “Terms and Conditions”, “Accessibility Statement”, “Freedom of Information” and “Crown copyright”.

View My Applications v1.1

View My Applications v1.1

This version incorporates all of the discussed changes to the design of VMA that includes headers and footers, a left hand menu, the search box, filtering, and an expanded entry etc.

The following changes were made in accordance with Sam Brierley's observations:

  • Added blue underline to the expandable rows. (To bring in line with the GOV Accordion pattern, and prompt user that the data is expandable).
  • Removed the "Customer Support" link from the header. (As this takes you to the customer support forum, it won't be specific to Portal (ie contextual help) and will break the user journey by opening a new window to a separate domain).
  • Removed the "Customer Support" link from the header. (As this takes you to the customer support forum, it won't be specific to Portal (ie contextual help) and will break the user journey by opening a new window to a separate domain).
  • Changed green text in the left hand menu to black text (in order to pass colour contrast accessibility checks).

View My Applications v1.2

View My Applications v1.2

This version incorporates all of the discussed changes to the design of VMA that includes headers and footers, a left hand menu, the search box, filtering, and an expanded entry etc.

The following changes were made in accordance with Sam Brierley's observations:

  • Added blue underline to the expandable rows. (To bring in line with the GOV Accordion pattern, and prompt user that the data is expandable).
  • Removed the "Customer Support" link from the header. (As this takes you to the customer support forum, it won't be specific to Portal (ie contextual help) and will break the user journey by opening a new window to a separate domain).
  • Changed green text in the left hand menu to black text (in order to pass colour contrast accessibility checks).

This version incorporates all of the discussed changes to the design of VMA that includes headers and footers, a left hand menu, the search box, filtering, and an expanded entry etc.

  • Agreed to retain "Customer Support" link after "Logout" within the header.
  • We agreed to retain the “Customer Support” link next to “Logout” in the VMA header. We discussed whether we should also add the “pop-out” window icon. It was felt that for the short term we add hover text “Opens in a new window”
  • Added hover text "Opens in a new window" instead of a pop-out icon.
  • We agreed to retain the four links in the footer and again for the short term we have the additional text below the links of “These links open in a new window”.

(The customer support and footer links would be revisited when the Portal Refresh light touch is being considered and developed).

It was agreed that the menu on the left hand side looks acceptable with the green text on white background, but not on a green hover tint. Again further insight is required as to make it fully accessible and clear that these are links without losing some of the clean appearance of the VMA service.

There was some significant discussion around how to indicate to customers that a parent row can be expanded to show additional details. It was suggested that the “+” sign on its own is insufficient and could fail accessibility requirements. It was also agreed that we should consider other options and that 1) form should follow function, 2) the bigger ‘hit area’ the better and 3) that the solution should pass accessibility. The initial suggestion was that there should be blue underlines as per the GDS recommendations, but this implied that the data may lead elsewhere, there was also suggestions around to add a “Tint on Hover” for the whole row.
